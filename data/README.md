# DeepOR Data

This directory contains sample data files for reproducing the DeepOR pipeline. Full datasets can be synthesized/generated using the provided code.

## Files

- **seed_problems.json** (9,132 instances)
  - Raw problem descriptions and ground-truth answers from the seed corpus.
  - Used as input to the Chain-of-Experts data synthesis pipeline.
  - Format per item:
    ```json
    {
      "instruction": "...",
      "input": "<problem description>",
      "output": "<solution code>",
      "answer": "<ground-truth objective value>"
    }
    ```

- **sft_data_sample.jsonl** (20 instances)
  - Sample of the supervised fine-tuning (SFT) dataset.
  - Follows the Alpaca format (`instruction` / `input` / `output` / `answer`).
  - The full dataset is synthesized by the CoE pipeline and then revised.

- **rl_data_sample.jsonl** (20 instances)
  - Sample of the reinforcement learning (GRPO) training dataset.
  - Format per item:
    ```json
    {
      "answer": "<ground-truth objective value>",
      "problem": "<problem description>"
    }
    ```

- **synthesis_examples/**
  - Four example outputs (`prob_0` ~ `prob_3`) from the data synthesis pipeline.
  - Each folder contains:
    - `description.txt`: The original problem description.
    - `cot.txt`: Raw Chain-of-Thought generated by Chain-of-Experts.
    - `revised_cot.txt`: Revised CoT that reads like a single expert's reasoning.
    - `answer.txt`: Ground-truth optimal objective value.

## How to Generate Full Data

### Step 1: Synthesize CoT data
```bash
cd data_synthesis
pip install -r requirements.txt
export LLM_API_KEY="your-api-key"
export LLM_MODEL="your-model-name"
python run_data_synthesis.py
```

### Step 2: Prepare SFT data
Convert the synthesized CoT outputs into the Alpaca format expected by LLaMA-Factory. See `expertise_tuning/README.md` for details.

### Step 3: Prepare RL data
Run the data preparation script to convert problems into the RL training format:
```bash
cd self_improvement
# Create rl_train.jsonl from your full problem set
# Format: {"problem": "...", "answer": "..."}
```
